# Security Policy

## Supported Versions

We only provide active security updates and patches for the **latest release** of the Clasp engine. Older major/minor versions will not receive backported security patches unless explicitly announced otherwise.

## Reporting a Vulnerability

Please do **NOT** open public GitHub issues or discussions for security bugs, vulnerabilities, or exploits. 

Instead, report vulnerabilities privately by emailing us directly at **security@openyf.ai**.

When submitting a report, please include:
- A detailed description of the vulnerability and its potential impact.
- Step-by-step instructions to reproduce the issue.
- Any relevant logs, code snippets, or proof-of-concept exploits.

### Our Commitment
We take all security reports seriously. Upon receiving your report, our team will:
- Acknowledge receipt of your vulnerability report within **48 hours**.
- Provide weekly progress updates to you until the vulnerability is resolved.

## Disclosure Policy

We follow a standard **90-day coordinated disclosure policy**.
This means:
1. We ask that you give us 90 days to develop, test, and release a patch before you publicly disclose the vulnerability.
2. We will work alongside you to safely coordinate public credit and disclosure once the fix has been shipped and downstream users have had a reasonable window to upgrade.
